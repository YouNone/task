Пользователь может просматривать все задачи в списке.
Пользователь может добавлять новые задачи.
Todo состоит из:
1 имя +
2 Описание + 
3 Информация о том, было ли выполнено задание +
4 Информация о том, является ли это особо важным «любимым» делом или нет +
5 Дата истечения срока +
6 email? +
Пользователь может удалять задачи. Удаление задачи необходимо подтвердить перед ее выполнением. +
Все информационные элементы, включая имя, должны быть изменяемыми после создания задачи.
Задачи должны храниться в кеше, который (включая все операции с кешем) представлен службой. ?
Если при создании или редактировании задачи поле пусто, отображается сообщение об ошибке,
 и процесс не продолжается.
Если изменить ввод поля электронной почты, 
сообщение об ошибке исчезнет сразу после ввода / удаления символа.
Задачи всегда следует сортировать по завершенным / незавершенным, 
а затем либо по важности + дате, либо по дате + важности.
Просроченные задачи - то есть задачи с истекшим сроком выполнения
 - должны быть визуально выделены.
